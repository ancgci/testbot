### Arquivo `requirements.txt`

Crie um arquivo chamado `requirements.txt` na raiz do seu repositório e adicione as seguintes linhas:

```
telethon
requests
```

### Explicação

- **telethon**: É a biblioteca que você está usando para interagir com a API do Telegram.
- **requests**: É a biblioteca que você pode usar para fazer requisições HTTP, caso precise.

### Exemplo Completo

Aqui está um exemplo completo de como o arquivo `requirements.txt` deve ficar:

```
# requirements.txt

# Dependências do projeto
telethon
requests
```

### Passo a Passo para Usar o `requirements.txt`

1. **Criar o arquivo `requirements.txt`**:
   - Crie um arquivo chamado `requirements.txt` na raiz do seu repositório.
   - Adicione as dependências conforme mostrado acima.

2. **Instalar as dependências**:
   - Abra o terminal ou prompt de comando.
   - Navegue até a pasta do seu projeto.
   - Execute o seguinte comando para instalar as dependências listadas no `requirements.txt`:

```bash
pip install -r requirements.txt
```

### Usando um Ambiente Virtual

É uma boa prática usar um ambiente virtual para gerenciar as dependências do seu projeto. Aqui estão os passos para criar e usar um ambiente virtual:

1. **Criar um ambiente virtual**:

```bash
python -m venv venv
```

2. **Ativar o ambiente virtual**:

   - No Windows:

```bash
.\venv\Scripts\activate
```

   - No macOS e Linux:

```bash
source venv/bin/activate
```

3. **Instalar as dependências**:

```bash
pip install -r requirements.txt
```

4. **Executar o script**:

```bash
python monitor_sol_trend.py
```

### Resumo
Para o seu projeto, o arquivo `requirements.txt` deve listar as dependências `telethon` e `requests`. Certifique-se de criar e ativar um ambiente virtual, e então instale as dependências usando o comando `pip install -r requirements.txt`. Isso garantirá que todas as dependências necessárias estejam instaladas no ambiente correto.
